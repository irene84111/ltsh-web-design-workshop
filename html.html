HTML自學秘笈

Hyper Text Markup Language

HTML是什麼？

HTML 是一種「標示語言」（ markup language ），用來描述網頁的內容，並在網頁瀏覽器（ IE, Chrome, Firefox, Safari 等）上呈現。

  • HTML 是「Hyper Text Markup Language」（ 超文字標示語言 ）的縮寫。
  • 標示語言由各種標籤（Tag）所組成。
  • 不同的標示語言有自己專屬的標籤，每個標籤都有自己的功能。

標籤（Tag）概念的建立

HTML也有自己專屬的標籤，不同的標籤有不同的功能和使用方法，活用標籤就能建立一個完善的網頁。

  • 每一個標籤都是由「<」及「>」所包圍，幾乎所有 HTML 的標籤都是成對的，例如：<p>文字</p>。
  • 以 <p>文字</p> 為例，前面的 < > 是開始標籤，後面的 </ > 是結尾標籤，結尾標籤必須加上一個「/」。
  • 被 < > 和 </ > 所包圍的內容會受到該組標籤的影響而呈現出不同的樣子和功能。
    包在 <h1> 和 </h1> 的文字，會變成一級標題；擺在 <b> 與 </b> 的文字會變成粗體。
  • 一組標籤可以包含另一組標籤，但要注意標籤的影響是有連帶關係的！
    <p>香蕉A<a href="">蘋果和香蕉</a>香蕉B</p>。
    　「蘋果和香蕉」會同時受到 <p> 和 <a> 的影響，而「香蕉A」和「香蕉B」只會受到 <p> 的影響。

HTML 基本架構

<!DOCTYPE html>
<html>
  <head>
    <title>Page Title</title>
  </head>

  <body>
    <h1>My First Heading</h1>
    <p>My first paragraph.</p>
  </body>
</html>
  • DOCTYPE 定義這個文件的類型是 HTML。
  • 在 HTML 中，可分為「頭」和「身體」兩部分：
    1. 「頭」的部分是由 <head> 和 </head> 標籤所界定，大部分用於規範和整體網頁相關的資訊，這些資訊通常不會直接呈現在網頁內。以上例來說，「頭」的部分包含了 <title>，用來說明此網頁的標題。
    2. 「身體」的部分是由 <body> 和 </body> 標籤所界定，就是網頁的內容，這些內容將直接呈現在網頁上。上例中的 <h1> 和 <p> 會把「My first Heading」和「My first paragraph.」呈現在網頁上。

網頁瀏覽器在讀取 HTML 檔案中的程式碼時，是由上往下一行一行讀取，在上面的程式碼會優先被執行。因此，在撰寫 HTML 時必須注意每個標籤的排序。如上例所示，由於 <h1> 在 <p> 的上一行，「My first Heading」也就會在「My first paragraph.」的上一行，如果把 <h1> 和 <p> 的順序對調，「My first paragraph.」就會在「My first Heading」上方。

養成撰寫 HTML 的良好習慣

仔細看看 HTML 程式碼所呈現的樣子。
當我們在撰寫 HTML 時，必須特別注意「分行」和「縮排」。雖然我們可以把所有程式碼全部擠在同一行，但這不僅會造成閱讀上的困難，也會讓之後在修改和除錯的時候非常不方便。

就讓我們來了解什麼是分行和縮排吧！

  • 不同的標籤通常會做分行的動作，例如 <h1> 和 <p> 不會出現在同一行。
  • HTML 中常常會出現「巢狀式結構」，即一組標籤內包含數個標籤。當標籤 A 裡面包含標籤 B 時，我們會視情況進行分行（以利於閱讀程式碼為主）。如果標籤 B 有換行，我們就要「縮排」。
    如上例所示，包含在 <body> 裡的 <h1> 和 <p> 都比 <body> 往前兩個半型空白，但結束的 </body> 和開始的 <body> 還是在同一列上，代表它們是一組的。
  • 常用的縮排是「兩個半型空白」或「一個 Tab」。
    但要避免在同一個 HTML 文件中混用這兩個方法。
  • 許多現行的文字編輯器（Sublime Text, Notepad, Textmate 等）會自動縮排，甚至有輔助線幫助對齊，但我們還是要時時檢查每個標籤是否有符合規定。
  • HTML 對於大小寫規定並不嚴格，<p> 和 <P> 是相同的，但我們還是以小寫為主。

文字(Texts)

  • Heading
  • Paragraph

Heading：<h1> ~ <h6>

<h#> 代表「標題」，共有 6 種，文字的大小和重要性從 <h1> 到 <h6> 遞減。

語法： 
<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>
網頁中呈現的樣子：
  • Heading1
  • Heading2
  • Heading3
  • Heading4
  • Heading5
  • Heading6
Heading 的重要性
  • Heading 標籤網頁的主要架構，也能讓使用者快速了解網頁的內容。
  • 只有在建立標題的時候才使用 Heading 標籤，請勿為了讓文字較大或粗體化而任意使用。
  • 現今的搜尋引擎（Google, Yahoo 等）會根據網頁的 Heading 標籤抓出網頁的架構和內容，任意使用會造成搜尋資料上的困難。
  • 網頁瀏覽器會自動在 Heading 標籤前後加上一點空白（margin）。


Paragraph：<p>

<p> 代表「段落」，用來呈現網頁的文字。

語法： 
<p>網頁最常使用的標籤</p>
網頁中呈現的樣子：

網頁最常使用的標籤

HTML 最後呈現出來的樣子會隨著瀏覽器的不同和視窗的大小而有所差異，即使是一模一樣的程式碼在不同的環境下看起來也可能不一樣。有一點需要特別注意，瀏覽器在讀取 HTML 程式碼時會自動忽略多餘的空白和行數，字元與字元之間最多只會保留一個半型空白！因此，在 <p> 內文 </p> 裡面留下過多的空白或換行都是沒用的。



Break：<br>

<br> 代表「中斷」，強迫文字在網頁上呈現時，能夠切換到下一列。<br> 不需要結尾標籤。

語法： 
<p>羅東高中<br>網頁<br>工作坊</p>
網頁中呈現的樣子：

羅東高中
網頁
工作坊



Horizontal Line：<hr>

<hr> 代表「水平線」，在網頁排版上很實用。<hr> 同樣不需要結尾標籤。

語法： 
<p>羅東高中網頁工作坊</p>
<hr>
<p>就在今年夏天！</p>
網頁中呈現的樣子：

羅東高中網頁工作坊

就在今年夏天！

Bold & Strong：<b> & <strong>

<b> 和 <strong> 都代表「粗體」，被這兩種標籤包圍的文字會以粗體呈現。
<b> 文字 </b> 、 <strong> 文字 </strong> 所呈現出來的效果是一樣的。

語法： 
<p> 羅東高中網頁工作坊 </p>
<br>
<b> 羅東高中網頁工作坊 </b>
<strong> 羅東高中網頁工作坊 </strong>
網頁中呈現的樣子：

羅東高中網頁工作坊

羅東高中網頁工作坊
羅東高中網頁工作坊


Italic & Emphasized：<i> & <em>

<i> 和 <em> 都代表「斜體」，被這兩種標籤包圍的文字會以斜體呈現。
<i> 文字 </i> 、 <em> 文字 </em> 所呈現出來的效果是一樣的。

語法： 
<p> 羅東高中網頁工作坊 </p>
<br>

羅東高中網頁工作坊

羅東高中網頁工作坊
羅東高中網頁工作坊

語法：

列表(List)

  • 無序列表（Unordered list）
  • 有序列表（Ordered list）


無序列表（Unordered list）

使用沒有排序意義的項目符號，常見如小黑點，我們也可以自行設定其他項目符號。
以 <ul> 定義列表、 <li> 定義列表中的每個項目，<li> 必須包覆在 <ul> 中。

語法： 
<ul>
  <li>羅東高中</li>
  <li>網頁工作坊</li>
  <li>LTSH Web Workshop</li>
</ul>
網頁中呈現的樣子：
  • 羅東高中
  • 網頁工作坊
  • LTSH Web Workshop


有序列表（Ordered list）

使用具有排序意義的項目符號，如數字或英文字母。
以 <ol> 定義列表、 <li> 定義列表中的每個項目，<li> 必須包覆在 <ol> 中。

語法： 
<ol>
  <li>羅東高中</li>
  <li>網頁工作坊</li>
  <li>LTSH Web Workshop</li>
</ol>
網頁中呈現的樣子：
  1. 羅東高中
  2. 網頁工作坊
  3. LTSH Web Workshop


如果我們在列表內再放入列表，就會形成巢狀式的列表，此時瀏覽器會自動縮排並且選換項目符號。
我們可以在這任一列表中任意置入另一種列表！只要符合列表的語法規定，就能做出所需要的列表。

語法： 
<ul>
  <li>羅東高中網頁工作坊
    <ol>
      <li>HTML</li>
      <li>CSS</li>
    </ol>
  </li>
  <li>資訊
    <ul>
      <li>時間：2015.7.20 ~ 21</li>
      <li>地點：羅東高中</li>
    </ul>
  </li>
  <li>LTSH Web Workshop</li>
</ul>
網頁中呈現的樣子：
  • 羅東高中網頁工作坊
    1. HTML
    2. CSS
  • 資訊
    • 時間：2015.7.20 ~ 21
    • 地點：羅東高中
  • LTSH Web Workshop

連結(Link)

連結是網頁中不可或缺的元素，讓我們可以迅速地連結網站中的其他分頁，甚至到全世界各個網頁。

語法： 
<a href="url"> 連結文字 </a>

href 代表這個連結指向的位置。將 url 替換成網頁的網址，就能連結到該網頁。

語法： 
<a href="http://d-d-x.github.io/ltsh/ "> 羅東高中網頁工作坊 </a>
網頁中呈現的樣子：
羅東高中網頁工作坊

當我們建立一個擁有數個分頁的網站時，我們也需要透過連結的方式將每個分頁串在一起。
將 url 替換成分頁的檔名，就能連結到該分頁。

語法： 
<a href="page.html"> 分頁 </a>

此時，「分頁」就會形成連結，連結到一個叫做「page.html」的網頁。

需要注意的是，兩個互相連結的 HTML 檔案必須在同一個目錄（資料夾）中，這樣 <a> 才找得到檔案的位置並建立連結，否則必須明確標示出檔案位置。


補充：絕對路徑 vs 相對路徑
  • 「相對路徑」就是相對於現在目錄的路徑表示法，因此「相對路徑」所指到的檔案或目錄，會隨著現在目錄的不同而改變。
    1. 「text.html」和「./text.html」是相同的，都表示同一層目錄下的 text.html 檔案。
    2. 「image/text.gif」表示 image 子目錄下的 text.gif 檔案。
    3. 「../index.html」表示上一層目錄下的 index.html 檔案。
    4. 「../html/page.html」表示上一層目錄下的 html 子目錄的 page.html 檔案。
  • 「絕對路徑」指的是一個絕對的位置，並不會隨著現在目錄的改變而改變。
    1. 「file:///d:/windows/page.html」表示本機 D 槽中的一個 HTML 檔案。
    2. 「http://d-d-x.github.io/ltsh/」表示網路上一台 WWW Server 上的一個 HTML 檔案。
  • 通常我們用「 . 」代表現在目錄，而用「 .. 」代表上一層目錄。
  • 一般而言，除非有特殊需求，否則在 HTML 文件中，我們都盡量用相對路徑來表示網址。
    使用相對路徑的好處是：當我們將包含許多網頁的目錄轉移到另一個位置時，雖然絕對位置改變，但是在此目錄下的網頁相對位置並沒有改變，因此我們並不需要大幅修改網頁內的網址。

影像(Images)

除了要能夠顯示文字外，網頁的另一個重要功能是要顯示影像。

語法： 
<img src="url" alt="影像無法呈現時所產生的文字">

如同設定連結，置入影像的方法也有兩種：

  1. 當我們需要影像時，常常會上 Google 搜尋。每個在網路上呈現的影像都有自己專屬的網址，我們可以直接將 url 替換成該影像的網址，就能在網頁中置入影像而不需要將影像下載到電腦裡，這也是絕對路徑的概念。
    • <img src="http://dgicdplf3pvka.cloudfront.net/images/dogbreeds/large/Welsh-Corgi-Pembroke.jpg" alt="這是一隻可愛的柯基犬">
  2. 當我們想使用自己電腦裡的影像時，就需要使用相對路徑。通常，我們會特別新增一個資料夾將網頁需要用到的影像全部放在裡面。透過相對路徑的設定，我們就能將影像呈現在網頁中。
    • <img src="images/corgi.jpg" alt="">
      表示在一個名為 image 的資料夾中， corgi.jpg 這個影像檔會被呈現在網頁中。

  • <img src="url" alt=""> 是少數在 HTML 不成對的標籤，因此沒有結尾標籤。
  • alt 可以設定「影像無法呈現時所產生的文字」，當圖片的位置錯誤或者失連，網頁就無法順利呈現該圖片，此時就會以 alt 代替，alt 也可以留白。
  • 配合 CSS 設定 img 樣式，可以讓影像在網頁上呈現的更漂亮。
  • 現今常見的影像檔類型如 .jpg, .png 或 .gif 都能順利呈現在網頁中，但置入影像時需要留意檔案的大小，解析度越高的影像需要花越久的時間讀取，會造成載入網頁的速度變慢。
  • 由於網頁是公開的，任何知道網址的人都可以連上特定的網頁。因此，在 HTML 置入影像時，必須注意智慧財產權！如果我們想要使用網路上的圖片，要事先確定是否有侵權的疑慮。

表格(Table)

HTML 在排版上除了使用框架之外，可以用更簡單的方法來完成，就是大家熟悉的表格。

語法： 
<table>
  <tr>
    <td> 第 1 列、第 1 行 </td>
    <td> 第 1 列、第 2 行 </td>
    <td> 第 1 列、第 3 行 </td>
  </tr>

  <tr>
    <td> 第 2 列、第 1 行 </td>
    <td> 第 2 列、第 2 行 </td>
    <td> 第 2 列、第 3 行 </td>
  </tr>
</table>
網頁中呈現的樣子：
第 1 列、第 1 行 第 1 列、第 2 行 第 1 列、第 3 行
第 2 列、第 1 行 第 2 列、第 2 行 第 2 列、第 3 行
  • <table> 用來定義整個表格。
  • <tr> 代表「table rows」，即表格「橫列」。
  • <td> 代表「table data」，即表格「直行」。
  • 在 HTML 中，我們會先定義表格的橫列再定義表格的直行。
  • 透過 CSS 設定 border 樣式就可以自由設計表格的邊框。

如果我們想要「合併儲存格」的時候，該怎麼做呢？

語法： 
<table>
  <tr>
    <td colspan="2"> 第 1 列、第 1 行 </td>
    <td> 第 1 列、第 2 行 </td>
  </tr>
  <tr>
    <td rowspan="2"> 第 2 列、第 1 行 </td>
    <td> 第 2 列、第 2 行 </td>
    <td> 第 2 列、第 3 行 </td>
  </tr>
  <tr>
    <td> 第 3 列、第 1 行 </td>
    <td> 第 3 列、第 2 行 </td>
  </tr>
</table>
網頁中呈現的樣子：
第 1 列、第 1 行 第 1 列、第 2 行
第 2 列、第 1 行 第 2 列、第 2 行 第 2 列、第 3 行
第 3 列、第 1 行 第 3 列、第 2 行

我們只需要對 <td> 標籤進行設定就可以合併儲存格：

  1. 在 <td> 標籤中加入「colspan="#"」就能合併橫列的儲存格。其中，2 < # < 表格中總橫列數
    （如果 # = 1 的話就沒有任何合併效果；如果 # 超過表格中總橫列數的話，效果等同於 # = 表格中總橫列數）
  2. 在 <td> 標籤中加入「rowspan="#"」就能合併直行的儲存格。其中，2 < # < 表格中總直行數
    （如果 # = 1 的話就沒有任何合併效果；如果 # 超過表格中總直行數的話，效果等同於 # = 表格中總直行數）

當我們使用 colspan 或 rowspan 進行合併時，實際上是以原有的儲存格向右或向下擴充。因此，需要反覆調整才能呈現最理想的樣子。

類別(Class) & ID

當我們在建立網頁時，會事先規劃要在網頁上呈現什麼內容。
以個人網頁為例，我們通常會放上自我介紹、頭像、興趣、專長、作品與聯絡方式等等。當許多不同類型的資料放在同一個頁面上，卻沒有任何分類，天呀！根本是一場災難！

  • Class
  • ID


什麼是類別（Class）？

相同類型的資料通常會套用相同的風格，並以不同的風格來區分不同類型的資料，如此一來就能讓網頁變得漂亮。 我們運用 CSS 來設定不同標籤的樣式，當我們在 HTML 中將相同類型的資料賦予同一個「類別（Class）」，並運用 CSS 中的 Class Selector 就能快速設定具有相同類別的資料樣式。

這樣有什麼好處呢？
  • 我們可以快速設定資料的樣式，提昇網頁設計的效率。
  • 避免不同類別的資料樣式互相干擾，降低網頁設計的困難度。
  • 讓程式碼更加清楚易讀，方便之後修改和除錯。

語法： 
<element class="value">
  • element 可以是除了base, head, html, meta, param, script, style 以及 title 以外的任意標籤，在該標籤下的任何標籤都會受到該類別樣式的影響。
  • value 就是類別的名字，可以自由命名。
舉例： 
<h1 class="blue"> Heading </h1>
<p class="red"> This is an example. </p>
<ul class="blue">
  <li> First </li>
  <li> Second </li>
</ul>
<p class="blue red"> This is another example. </p>
  • 我建立了兩個類別分別是 blue 和 red，<h1> 和 <ul> 下的所有 <li> 都會受到 blue 類別樣式的影響，第一個 <p> 只會受到 red 類別樣式的影響。
  • 一個標籤可以有好幾個類別嗎？可以！就像第二個 <p> 同時會受到 blue 和 red 類別樣式的影響，在設定時只要以一個半型空白將不同類別分開即可。但這時後必須注意 blue 和 red 類別是否有互相衝突的樣式設定，這可能會導致 <p> 標籤無法正常顯示。


什麼是 id？

id 的作用和類別很像，都可以透過 CSS 設定專屬的樣式並套用在 HTML 中的特定標籤上。
但 id 和類別最大的不同在於，id 是唯一的！就像每個人都有自己專屬且唯一的身分證字號一樣，id 在 HTML 只能出現一次，等於只能作用在一個標籤上。由於這樣的限制，除了在開發 Javascript 時會較常使用之外，id 的使用率通常比類別低。

語法： 
<element id="value">

用法和類別一樣。

另外，id 也可以用來作為「連結錨（link anchor）」。


什麼是連結錨（link anchor）？

有時候我們點擊網頁中某個連結或按鈕時，會跳到同一個網頁中的某個位置，我們可以透過這樣的方式設定快速連結，方便使用者迅速地在網頁中找到想看的東西。

語法： 
<h1 id="one"> Heading </h1>
<a herf="#one"> Anchor </a>
  • 設定連結錨（link anchor）之前，我們要決定哪個標籤要被作為連結目標。在這裡，我以 <h1> 作為目標並設定 id = "one"。此時，「Heading」這段文字就有 one 這個 id 了。
  • 使用連結的語法，並定義 herf = "#one"。其中，# 是 id 的選擇器，任何 id 前面都需要加上 #，HTML 才會知道這個連結要尋找的對象是一個 id。此時，當我們點擊「Anchor」這段文字時，網頁視窗就會跳到「Heading」的位置。

連結錨的作用只限於同一個網頁之下，如果要連到其他網頁去就必須遵循一般的連結方式，忘記了就來複習吧！


在命名 class 或 id 時，請遵守以下命名規則：
  1. 一律使用小寫英文。
  2. 不能以數字開頭。
  3. 不能包含空白、底線等任何非英文字母的特殊字元。
  4. 名稱盡量符合類別樣式的特色，有利於閱讀時可以馬上知道該類別的樣式。

區塊(Division)

前面我們提到「類別（Class）」可以用來分類不同的資料。只要將不同的標籤賦予相同的類別，就能獲得相同的樣式。
假設我正在做自己的個人網站，在自我介紹的部份，我使用了 <h1>、<p>、<ul> 還有許多 <a>，當然還要使用 <img> 放上自己的頭像。接著我建立了一個類別叫做 intro。天呀！現在我需要把標籤一個一個打上類別嗎？

這時候，我們就需要使用 <div> 標籤：

<div> 用來定義 HTML 中的區塊（division），它就像一個盒子一樣，可以把相同類型的標籤放在一起。 以個人網站為例，我打算將網頁切成「自我介紹」、「興趣」、和「聯絡方式」等三個區塊。每個區塊可以用 <div> 區分開來。

語法： 
<div class="myself">
  <h1> About Me </h1>
  <p> Hi! 我是 Irene，很高興認識你！</p>
  <img src="images/irene.jpg" alt="">
</div>

<div class="interest">
  <h1> 我的興趣 </h1>
  <ul>
    <li> 網頁設計 </li>
    <li> 看電影 </li>
  </ul>
</div>

<div class="contact">
  <h1> 聯絡方式 </h1>
  <a herf="https://www.facebook.com/irene.chang.735"> Facebook &tl;/a>
</div>
  • 看著程式碼，你可以很快地找出三個區塊，每個區塊都包覆在一個 <div> 標籤裡。善用 <div> 標籤可以讓程式碼變得容易閱讀和修改。
  • 每個 <div>都賦予不同的類別（class），這樣的方式可以讓同一個區塊下的所有標籤獲得相同的樣式，比起一個一個設定類別，直接設定 <div> 的類別可以大大提昇網頁設計的效率。

註解(Comment)

撰寫 HTML 的時候，我們常常需要做點小筆記，可能是針對某些區塊特別作說明，或是某些部分還不完整，想要以後再回來修正。尤其當一個網頁擁有上百行的程式碼，或是好幾個人一起設計同一個網頁時，「註解」就非常重要。

語法： 
<!-- 註解文字 -->
舉例： 
<p> 羅東高中網頁工作坊 </p>
<!-- 要記得加上連結 -->
<!-- 註解文字不會呈現在網頁上 -->
網頁中呈現的樣子：

羅東高中網頁工作坊

  • 註解只有網頁製作者看得到，可以放在任何位置而不會對網頁造成任何影響，網頁瀏覽器在讀取 HTML 文件時會自動跳過註解而不會呈現在網頁上。
  • 當我們有東西已經寫入 HTML 但還不想呈現在網頁上時，我們可以直接使用註解而不需要刪除，想要呈現在網頁上的時候，只要將註解標籤拿掉就可以了。
  • 善用註解可以提昇網頁製作的效率！
